Idea 1: 

This UI design is a single-screen energy tracking dashboard organized into a clean vertical layout. At the top, a header displays navigation arrows alongside the day and date to let users easily switch views. The main middle section places a horizontal daily budget progress bar directly next to a vertical column graph. This graph is split into off-peak and peak time slots, featuring dynamic bars that change color from red to green depending on energy usage.

Pros: 
- Shows energy data clearly througfhoout the day, simutaneously teaching the user about the peak and offpeak times in a clear manner 
- Daily budget is shown in a simplistic manner 

Cons: 
- traditional Column graphs can possibly be overwhelming to users, because it is a lot of data concentrated into one place
- The simplified budget bar could mean that the users attention is drawn away from the budgeting aspect of the app, and focus more onn the data, which defeats the whole point. 

Idea 2:  

This alternative UI layout features a top header displaying the day, date, and year with arrows for easy timeline navigation. The primary focal point is a large, segmented semi-circular dial enclosing text placeholders for the current day's cost and total energy usage in kilowatt-hours. Positioned next to this dial is a subtle, unlabelled column graph that tracks consumption patterns across the entire day from 12 AM to 11:59 PM. This smaller chart includes a bottom timeline broken into off-peak and peak segments to show exactly when the energy was used. This graph at the bottom must be much smaller and discrete.

Pros: 
- Shows budget as  primary foucs 
- Time graph below is subtle, giving the user the info but not distracting from the budgeting feature 

Cons: 
- Info may be hard to read on the graph
