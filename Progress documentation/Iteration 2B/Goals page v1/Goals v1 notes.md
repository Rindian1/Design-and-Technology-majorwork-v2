Goals page has been implemented 

What im happy with: 
    - The core functionality works, the goals are toggle-able, the stakes remove points when activated, and the points are added back when deactivated 
    - The overall aesthetics are clear and very easy to understand 

What I am going to change   
    - Upon further inspection, I dont beleive a stakes feature in general alligns with the design brief. Stakes would likely just discourage users from selecting and opting into goals in the first place, which is not really worth the hypothetical raise in investment that a user will have in the app. It would be better that the stake feature is removed, and keep it simplified to just gaining points.

    - Currently, I am not sure how to present the goals tab as whole. Right now, in the eyes of the markers, all they can see is that they click on the goals, but they can not *see* the gainage of points, as the database is static

    - Add a feature so that when a user claims their points, some kind of satisfying effect comes up to signify their achievment. The user should be able to claim their points for each day increment.  

    - Make it so that the points are visible from the menu bar, as the user should be able to be driven by it consistently. Should be placed at the top of the home screen, that should double as a button to redirect the user back to the goals page
